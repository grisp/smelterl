# Task 4.4b: `smelterl_gen_external_desc` template-owned formatting refactor

## Outcome

Updated `smelterl_gen_external_desc` and `priv/templates/external.desc.mustache`
so the generator now passes structured `description`/`version` fields plus
simple presence flags into the shared Mustache renderer instead of assembling a
final description string in Erlang. The emitted `external.desc` output stays
compatible with the previous implementation while moving the optional version
separator and version-only formatting into the template.

## Key Decisions

- Kept product-name normalization in Erlang as required by the task, but moved
  the final `desc:` line shape into the template using section flags
  (`has_description`, `has_version`) rather than a preformatted `desc`
  string.
- Reused the existing section-capable `smelterl_template` renderer instead of
  extending the template language further; the current section support is
  enough to express all four output combinations.
- Added focused Common Test coverage for description-only, version-only, and
  empty-metadata cases so the template-owned formatting branches are explicit
  rather than being inferred only from the original happy-path case.
