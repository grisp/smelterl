# Task 4.4a: `smelterl_gen_config_in` template-owned formatting refactor

## Outcome

Updated `smelterl_gen_config_in` and `priv/templates/Config.in.mustache` so
the generator now passes structured extra-config entries and package/source
sections into the shared Mustache renderer instead of assembling formatted
`config` and `source` blocks in Erlang. The emitted `Config.in` output stays
byte-for-byte compatible with the previous generator behavior while shifting
comments, blank lines, and line shape fully into the template.

## Key Decisions

- Reused the same section-driven template contract introduced by Task 4.4 for
  `external.mk`: Erlang keeps deterministic ordering and package discovery,
  while the template owns presentation details.
- Modeled `Config.in` package groups as structured per-nugget maps
  (`name`, `description`, `sources`) rather than comment/source iodata blocks,
  which keeps the generator aligned with the existing `external.mk` data shape
  and makes later template-only formatting changes straightforward.
- Added a focused Common Test case for an empty nugget description so the
  template-owned `# name` vs `# name: description` formatting branch is
  covered explicitly.
