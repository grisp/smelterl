# Task 3.2b - Smelterl Appendix B formatting and documentation conformance

## Objective

Bring the current Smelterl Erlang module set into conformance with
`docs/02_SMELTERL_DESIGN.md` Appendix B and §5.3 by adding the required
REUSE/SPDX headers, module/function/type documentation, and the documented
source-file section structure, including explicit two-space indentation for
wrapped function-head `when` guards, without changing command behavior.

## Design Grounding

- Read `AGENTS.md`, `docs/00_OVERVIEW.md`, `docs/01_DATA_DESIGN.md`,
  `docs/02_SMELTERL_DESIGN.md`, `docs/03_ALLOY_DESIGN.md`,
  `docs/WORKFLOW.md`, `docs/PLANNING.md`, and `scripts/tests/README.md`.
- Focused design sections:
  - `docs/02_SMELTERL_DESIGN.md` §5.3 Documentation
  - `docs/02_SMELTERL_DESIGN.md` Appendix B - Format and Documentation

## Initial Investigation

- The current `smelterl/src/*.erl` modules compile and test, but they do not
  yet follow the documented Appendix B structure.
- Missing pieces include SPDX/REUSE headers, `-moduledoc` text, `-doc`
  coverage for exported functions/types/callbacks, and the required section
  headers and export layout.

## Key Decisions

- Kept the task scoped to the production modules under `smelterl/src/`, which
  is the source set referenced by the backlog item and the part of the tree
  that defines the public command/runtime contracts.
- Added full Appendix B scaffolding to each touched module:
  - REUSE/SPDX headers
  - `-moduledoc`
  - `-doc` coverage for exported types and behaviour callbacks
  - section headers in the documented order
  - one `-export([...]).` line per exported function
- Clarified Appendix B itself to state that when a function head wraps onto
  multiple lines and the guard moves to the next line, `when` is indented by
  two spaces as a continuation of the head.
- Added explicit preferred vs not-preferred Appendix B examples so future
  formatting passes do not have to infer the wrapped-guard indentation rule
  from prose alone.
- Preserved behavior by avoiding parser or loader logic changes beyond
  formatting and documentation-only rewrites.
- Treated `smelterl_cmd_plan` exports as behaviour callback implementations:
  the contract documentation lives in `smelterl_command`, while the command
  module itself documents the module-level purpose and keeps callback
  implementations uncluttered.

## Residual Risks

- The Common Test suites under `smelterl/test/` still use the older file
  layout/documentation style. This task intentionally focused on the runtime
  source modules, so a later repo-wide REUSE/documentation sweep may still want
  to align test modules as well if that becomes a project-wide requirement.

## Planning Refinement

- Added a refinement note to `Task 3.3` so new Smelterl Erlang modules start
  from the Appendix B-compliant skeleton established here instead of requiring
  another cleanup pass after more pipeline modules land.
