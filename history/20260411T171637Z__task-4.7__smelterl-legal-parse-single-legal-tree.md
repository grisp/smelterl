# Task 4.7: `smelterl_legal` parse single legal tree

## Objective

Implement reusable parsing of one Buildroot `legal-info/` tree so later
Smelterl generate-stage legal merge/export work can consume structured legal
data instead of reparsing Buildroot output ad hoc.

## Outcome

Added `smelterl_legal:parse_legal/1`, shared `br_package_entry()` and
`br_legal_info()` types in `smelterl.erl`, and focused Common Test coverage
for successful package extraction plus invalid-path, missing-manifest, and
malformed-CSV failures. The parser now reads one Buildroot `legal-info/`
directory, extracts target packages and host packages from `manifest.csv` and
`host-manifest.csv`, promotes the special `buildroot` host-manifest row to the
top-level `br_version` field instead of keeping it in `host_packages`, accepts
current Buildroot synthetic host rows with empty `VERSION`, tolerates bare
quotes inside quoted license fields as emitted by some packages, and resolves
`license_files` to actual paths relative to the `legal-info/` root.

## Key Decisions

- Kept parsing header-driven instead of hard-coding column positions beyond the
  required field names (`package`, `version`, `license`, `license files`), so
  later merge/finalize work can tolerate Buildroot adding extra manifest
  columns without changing the single-tree parser.
- Allowed empty `version` values because current Buildroot host manifests emit
  synthetic entries such as `go-src` and `go-bin` with no version string, and
  the parser should reflect actual legal-info exports rather than reject them.
- Resolved `license_files` against the on-disk `licenses/` or `host-licenses/`
  tree so the returned paths are usable relative to the `legal-info/` root as
  required by the design. The normal case is `<name>-<version>/...`, but the
  parser also supports Buildroot's `buildroot` and empty-version host-package
  directories that are named only by package id.
- Normalized `license files` values of `not saved`, `unknown`, or empty to an
  empty list because those markers do not correspond to concrete files under
  the exported `legal-info/` tree.
- Folded the `buildroot` row from `host-manifest.csv` into `br_version` and
  excluded it from `host_packages`, keeping later manifest/export consumers
  from having to special-case that pseudo-package repeatedly.
- Made the CSV parser intentionally permissive for bare quotes inside quoted
  fields because the real Buildroot sample contains license strings such as
  `BSD 3-Clause "New" or "Revised" License` without CSV escaping.
