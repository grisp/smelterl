## Objective

Centralize shared cross-module Smelterl Erlang types in `smelterl.erl`, update
duplicate module-local type declarations to remote type references, and make
the source-of-truth rule explicit in both the Smelterl design doc and the AI
workflow.

## Key Decisions

- Kept `smelterl.erl` as the single exported source of cross-module shared
  Erlang types and converted the duplicated plan-pipeline modules to remote
  type references rather than maintaining parallel local aliases.
- Left module-private helper types such as validator-local maps and context
  shapes in their owning modules, because they are not shared contracts.
- Updated both the Smelterl design doc and the workflow doc in the same task so
  the source-of-truth rule is explicit in process as well as code.
