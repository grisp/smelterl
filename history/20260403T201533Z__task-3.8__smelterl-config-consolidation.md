## Key Decisions

- Implemented config consolidation as a dedicated `smelterl_config` module
  that consumes overridden target-local motherlodes directly, so later stages
  do not need to reconstruct origin metadata or reapply overrides.
- Reused validator-backed flavor resolution via a new
  `smelterl_validate:resolved_flavors/2` helper instead of duplicating flavor
  constraint logic inside the config resolver.
- Normalized `--extra-config` during `smelterl plan`, rejected user attempts to
  set `ALLOY_MOTHERLODE`, and injected the reserved
  `<<"${ALLOY_MOTHERLODE}">>` template value centrally so later plan/generate
  stages can consume one canonical extra-config map.

## Unexpected Issues And Resolution

- Erlang/OTP on this repository rejected `binary:split/3` with the attempted
  `{parts, 2}` option during focused tests. Replaced both parsing call sites
  with explicit `binary:match/2`-based split logic to keep the implementation
  compatible with the current toolchain.
- `rebar3` regenerated an untracked `smelterl/rebar.lock` during validation.
  It was removed before finalizing because this task did not change
  dependencies.
