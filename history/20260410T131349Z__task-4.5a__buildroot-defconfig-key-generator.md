## Task

Task 4.5a: Buildroot-driven `defconfig-keys.spec` generator.

## Objective

Replace the hand-maintained first draft `priv/defconfig-keys.spec` workflow
with a traceable generator that scans a Buildroot source tree, applies
conservative heuristics plus explicit include/override options, and produces a
committable spec file without embedding host-specific absolute paths.

## Outcome

Added a self-contained `scripts/generate_defconfig_keys.escript` and Common
Test coverage, replacing the earlier wrapper-plus-module split. The helper was
then used to regenerate `priv/defconfig-keys.spec` from the Buildroot 2025.05
checkout. The generated spec now carries a traceability header with Buildroot
version/revision, explicit include/override settings, and a regeneration
command that uses only sanitized path labels.

## Key Decisions

- Kept the generator conservative by design: a key is emitted automatically
  only when the Buildroot Kconfig help/name patterns strongly indicate both
  cumulative semantics and the `path` vs `plain` value kind.
- Kept the helper self-contained instead of runtime-compiling a separate
  application module. The script is only a developer maintenance tool, so
  portability and low operational friction matter more here than preserving a
  separately compiled module interface.
- Added explicit `--include` and `--override KEY=path|plain` options instead
  of trying to encode every Buildroot edge case into heuristics. Forced
  includes fail when the generator still cannot infer a kind, so maintainers
  have to make that decision explicit.
- Sanitized the traceability header to avoid leaking host-specific absolute
  paths. The committed file records Buildroot version/revision and a
  regeneration command that uses the Buildroot directory basename rather than
  the full local source path.
- Updated the Smelterl design document to make the generated spec workflow the
  documented source of truth instead of treating `priv/defconfig-keys.spec` as
  an ad-hoc manually curated file.

## Validation

- Focused baseline Common Test:
  `rebar3 as test ct --suite=test/smelterl_buildroot_defconfig_keys_SUITE`
- Helper CLI sanity:
  `escript scripts/generate_defconfig_keys.escript --help`
- Full Common Test:
  `rebar3 as test ct`
- Dialyzer:
  `rebar3 dialyzer`

## Residual Risks

- The generator remains intentionally conservative, so some valid cumulative
  Buildroot keys may still require explicit `--include` or `--override`
  options until future tasks expand the heuristics.
- The first committed generated index is based on the Buildroot 2025.05 tree
  available during this task; other Buildroot versions may expose additional
  cumulative keys that will require regeneration when Alloy moves to a new
  baseline.
