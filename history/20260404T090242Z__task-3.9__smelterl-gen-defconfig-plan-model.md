## Task

Task 3.9: `smelterl_gen_defconfig` plan-stage model build.

## Objective

Implement the plan-stage defconfig model builder described in `docs/02_SMELTERL_DESIGN.md`, add focused tests for cumulative-key handling and wrapper-hook injection, and integrate the model build into the current `smelterl plan` pipeline without extending scope into later serialization/render tasks.

## Outcome

Added `smelterl_gen_defconfig` and wired `smelterl plan` to build a defconfig
model for every target after config consolidation. The new plan-stage model
supports flavor-mapped fragment selection, single-pass `[[KEY]]` substitution
from consolidated config plus normalized extra-config, deterministic
regular/cumulative merge behavior, path resolution for cumulative path keys,
and automatic target-local wrapper hook injection.

## Key Decisions

- Kept the task strictly plan-stage: `smelterl plan` now builds the models and
  validates their inputs, but still leaves plan serialization and generate-time
  rendering to the existing follow-up backlog tasks.
- Stored cumulative-key policy in `smelterl/priv/defconfig-keys.spec` so the
  merge behavior is data-driven rather than hard-coded in the planner.
- Merged normalized extra-config entries into the target config maps only for
  defconfig substitution, reusing the existing `smelterl:config_entry()`
  shape instead of inventing a parallel substitution structure.
- Updated the flaky Common Test temp-dir helper in
  `smelterl_config_SUITE` after the full gate exposed a rerun collision in the
  old one-shot `file:make_dir/1` helper.
