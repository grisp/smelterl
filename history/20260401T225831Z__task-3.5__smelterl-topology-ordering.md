# Task 3.5 - smelterl_topology deterministic ordering

## Objective

Implement `smelterl_topology` so Smelterl can compute a deterministic
topological order for each validated target tree, with dependency-before-
dependent ordering that is stable across repeated runs.

## Initial Investigation

- `Task 3.3` already records ordered nugget dependencies in the tree edges.
- `Task 3.4` and `Task 3.4b` now validate targets before later plan stages and
  introduced Appendix B guidance to keep new Erlang control flow readable.
- Baseline validation must pass before implementation starts.

## Key Decisions

- Added a dedicated `smelterl_topology` module rather than burying ordering
  logic in `smelterl_tree` or `smelterl_cmd_plan`, so later pipeline stages can
  reuse one canonical deterministic ordering function.
- Used DFS postorder over the existing tree edges instead of reconstructing a
  separate Kahn queue, because the tree already stores dependency declaration
  order and DFS naturally preserves that order as the stable tie-break while
  keeping dependencies before dependents and the root last.
- Kept cycle detection in `smelterl_topology` even though validated trees
  should already be acyclic. This preserves the module’s standalone contract
  and makes its direct tests meaningful.
- Wired `smelterl plan` through per-target topology computation (main plus each
  auxiliary target) before the remaining pipeline stub so the new stage is
  exercised in the command flow rather than existing only as an isolated
  library.

## Residual Risks

- The current tree shape stores dependency declaration order only at the edge
  list level, so future override/remap work must refresh the full target tree
  before reordering if any dependency list changes.
- The command path now computes topology for all targets but does not yet
  persist those orders anywhere because later plan serialization tasks are not
  implemented yet.

## Planning Refinement

- Added a refinement note to `Task 3.6` that override/remap work must
  recompute impacted topology orders with `smelterl_topology`, preserving the
  dependency-order tie-break and root-last contract.
