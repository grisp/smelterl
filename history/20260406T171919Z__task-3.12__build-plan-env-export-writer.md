## Task

Task 3.12: `build_plan.env` export writer.

## Objective

Implement `build_plan.env` output for `smelterl plan`, using the serialized
plan payload as the source of truth for ordered auxiliary loop metadata and the
normalized root-level extra-config map, and cover the result with focused
golden-style tests.

## Outcome

Completed the optional `--output-plan-env` path in `smelterl plan`. Successful
plan runs now write a sourceable bash summary alongside `build_plan.term`,
carrying deterministic target-loop metadata and the normalized plan-level
`extra_config` payload.

## Key Decisions

- Kept `build_plan.env` in `smelterl_plan` so both the authoritative term file
  and its bash-friendly summary are produced by one serialization layer.
- Used the serialized `auxiliary_ids` list as the only source of auxiliary
  ordering, then appended `main` last for `ALLOY_PLAN_TARGET_IDS`, so shell
  consumers never depend on Erlang map iteration order.
- Emitted bash indexed arrays for loop order and associative arrays for
  per-target/root-level metadata because the Alloy workflow already assumes
  bash 4+ and sourcing, not POSIX shell export semantics.
- Single-quoted all rendered values so placeholder-style extra-config values
  like `${ALLOY_MOTHERLODE}` remain literal when the file is sourced.
